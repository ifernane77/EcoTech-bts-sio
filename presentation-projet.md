# Présentation du projet — EcoTech Suivi Chantier

## Contexte

EcoTech est une entreprise située à Saint-Pierre, à La Réunion.

Elle intervient dans les domaines du génie civil, du BTP, de la rénovation et de la construction. Elle travaille avec des particuliers, des entreprises et des promoteurs.

L’entreprise réalise notamment :

- du suivi de chantier ;
- des devis ;
- des plans ;
- des métrés ;
- du terrassement ;
- des fondations ;
- des travaux de maçonnerie.

## Problème constaté

Les informations liées aux chantiers sont actuellement réparties entre plusieurs outils :

- Google Drive ;
- les ordinateurs de l’entreprise ;
- les emails ;
- WhatsApp.

Les documents ne sont pas forcément perdus, mais l’équipe peut perdre du temps à les retrouver.

L’entreprise manque également d’une vue centralisée sur l’avancement des chantiers.

## Objectif du projet

Le projet EcoTech Suivi Chantier consiste à développer une application web simple permettant de centraliser les informations importantes liées aux chantiers.

L’application doit faciliter :

- la gestion des utilisateurs ;
- la gestion des clients ;
- la gestion des chantiers ;
- le stockage des documents ;
- l’ajout de photos ;
- le suivi des interventions ;
- la consultation de l’avancement.

## Utilisateurs concernés

L’application pourra être utilisée par :

- les gérants ;
- les employés administratifs ;
- les employés chargés du suivi des chantiers.

Les clients ne disposent pas d’un accès à l’application.

## Fonctionnalités principales prévues

- authentification ;
- gestion des rôles ;
- gestion des clients ;
- gestion des chantiers ;
- ajout de documents PDF ;
- ajout de photos ;
- historique des interventions ;
- génération d’un rapport PDF simple ;
- contrôle des droits selon le rôle.

## Stockage actuel

Avant la mise en place de l’application, les informations sont stockées dans plusieurs endroits :

| Outil | Utilisation |
|---|---|
| Google Drive | Documents partagés |
| Ordinateurs personnels | Fichiers de travail |
| Emails | Envoi de documents |
| WhatsApp | Messages et photos de chantier |

L’application doit permettre de centraliser progressivement ces informations.

## Limites du projet

Le projet reste volontairement adapté au niveau BTS SIO SLAM.

Il ne prévoit pas :

- d’accès client ;
- de facturation automatique ;
- de création complète de devis ;
- d’application mobile native ;
- de gestion comptable ;
- de fonctionnalités complexes inutiles pour l’épreuve.

Les devis et les factures seront stockés comme documents joints.
