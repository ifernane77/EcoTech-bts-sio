# Expression du besoin — EcoTech Suivi Chantier

## Besoin principal

L’entreprise EcoTech souhaite disposer d’une application web interne pour centraliser les informations liées au suivi des chantiers.

Actuellement, les informations sont dispersées entre Google Drive, les mails, WhatsApp et les ordinateurs personnels. Cette situation entraîne une perte de temps lorsqu’un membre de l’équipe doit retrouver un document, une photo ou une information liée à un chantier.

## Objectifs fonctionnels

L’application devra permettre de :

- gérer les utilisateurs ;
- gérer les rôles ;
- gérer les clients ;
- gérer les chantiers ;
- ajouter des documents PDF ;
- stocker les devis et factures comme documents joints ;
- ajouter des photos de chantier ;
- enregistrer les interventions ;
- générer un rapport PDF de chantier ;
- limiter les accès selon le rôle de l’utilisateur.

## Utilisateurs concernés

Les utilisateurs prévus sont :

- administrateur ;
- gérant ;
- employé.

Il n’est pas prévu de créer un accès client pour la première version de l’application.

## Contraintes

L’application doit :

- rester simple ;
- être démontrable en local ;
- être compréhensible par un jury BTS SIO ;
- utiliser une stack technique accessible ;
- être documentée progressivement ;
- produire des preuves exploitables pour le portfolio.

## Critères de réussite

Le projet sera considéré comme réussi si :

- les chantiers peuvent être créés et consultés ;
- les clients peuvent être associés aux chantiers ;
- les documents et photos peuvent être ajoutés ;
- les interventions peuvent être historisées ;
- les rôles utilisateurs sont pris en compte ;
- un rapport PDF peut être généré ;
- le projet est correctement documenté sur GitHub.
