# Backlog — EcoTech Suivi Chantier

## Objectif du backlog

Ce backlog permet de suivre les fonctionnalités prévues pour le projet EcoTech Suivi Chantier.

Il sera mis à jour progressivement pendant l’avancement du projet.

## Fonctionnalités prioritaires

| Priorité | Fonctionnalité | Description | Statut |
|---|---|---|---|
| Haute | Authentification | Permettre aux utilisateurs de se connecter | À faire |
| Haute | Gestion des rôles | Prévoir les rôles admin, gérant et employé | À faire |
| Haute | Gestion des clients | Ajouter, modifier et consulter les clients | À faire |
| Haute | Gestion des chantiers | Créer et suivre les chantiers | À faire |
| Haute | Documents PDF | Ajouter des documents liés aux chantiers | À faire |
| Haute | Photos de chantier | Ajouter des photos pour suivre l’avancement | À faire |
| Moyenne | Historique des interventions | Garder une trace des interventions réalisées | À faire |
| Moyenne | Rapport PDF | Générer un rapport de chantier | À faire |
| Basse | Amélioration graphique | Améliorer l’affichage avec Bootstrap | À faire |

## Version minimale attendue

La version minimale doit permettre de :

- se connecter ;
- consulter une liste de chantiers ;
- créer un chantier ;
- associer un client à un chantier ;
- ajouter un document ou une photo ;
- enregistrer une intervention.

## Version améliorée

La version améliorée pourra permettre de :

- générer un rapport PDF ;
- filtrer les chantiers ;
- améliorer les droits selon les rôles ;
- améliorer l’interface utilisateur.
