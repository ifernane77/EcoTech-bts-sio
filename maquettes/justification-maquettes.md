# Justification des maquettes — EcoTech Suivi Chantier

## Objectif

Ce document explique les choix réalisés pour les maquettes de l’application EcoTech Suivi Chantier.

---

## Choix général

Les maquettes sont simples, car l’objectif est de créer une application réaliste pour le BTS SIO SLAM.

L’application doit être compréhensible rapidement par les utilisateurs d’EcoTech.

---

## Choix des écrans

| N° | Écran | Justification |
|---:|---|---|
| 1 | Page de connexion | Sécuriser l’accès à l’application |
| 2 | Liste des chantiers | Donner une vue globale sur les chantiers |
| 3 | Fiche chantier | Centraliser les informations principales |
| 4 | Documents et photos | Regrouper les fichiers et preuves chantier |
| 5 | Historique des interventions | Suivre les actions réalisées |
| 6 | Rapport PDF | Produire une synthèse exploitable |
| 7 | Administration des utilisateurs | Gérer les rôles et les accès |

---

## Simplicité volontaire

Certaines fonctionnalités ne sont pas prévues dans la première version :

- accès client ;
- messagerie interne ;
- notifications automatiques ;
- application mobile ;
- paiement en ligne ;
- synchronisation Google Drive.

Ces choix permettent de garder un projet terminable et présentable au BTS.

---

## Lien avec le besoin EcoTech

Le problème principal d’EcoTech est la dispersion des informations.

Les informations sont réparties entre :

- Google Drive ;
- PC personnels ;
- mails ;
- WhatsApp ;
- échanges internes.

Les maquettes répondent à ce problème en regroupant :

- les chantiers ;
- les clients ;
- les documents ;
- les photos ;
- les interventions ;
- les rapports.

---

## Lien avec la suite du projet

Ces maquettes serviront à préparer :

- les pages PHP ;
- les formulaires ;
- les contrôles d’accès ;
- les tests fonctionnels ;
- la démonstration devant le jury.

---

## Conclusion

Ces maquettes permettent de visualiser l’application avant le développement.

Elles constituent une preuve de conception pour l’épreuve E6 SLAM.
