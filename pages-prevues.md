# Pages prévues — EcoTech Suivi Chantier

## Objectif

Ce document liste les pages prévues pour la future application EcoTech Suivi Chantier.

À cette étape, les pages ne sont pas encore développées.

Le but est de préparer les futurs écrans avant le développement.

---

## Pages principales prévues

| N° | Page prévue | Rôle |
|---:|---|---|
| 1 | Connexion | Permettre aux utilisateurs internes de se connecter |
| 2 | Tableau de bord | Afficher une vue globale des chantiers |
| 3 | Liste des clients | Consulter les clients enregistrés |
| 4 | Fiche client | Voir les informations d’un client |
| 5 | Liste des chantiers | Consulter les chantiers |
| 6 | Fiche chantier | Centraliser les informations d’un chantier |
| 7 | Documents et photos | Gérer les documents PDF et les photos |
| 8 | Interventions | Suivre l’historique des interventions |
| 9 | Rapport PDF | Préparer une synthèse de chantier |
| 10 | Administration utilisateurs | Gérer les comptes et les rôles |

---

## 1. Page de connexion

Cette page permettra à un utilisateur interne de se connecter.

Utilisateurs concernés :

- administrateur ;
- gérant ;
- employé.

---

## 2. Tableau de bord

Cette page affichera une vue globale de l’activité.

Exemples d’informations prévues :

- nombre de chantiers en cours ;
- nombre de chantiers terminés ;
- derniers documents ajoutés ;
- dernières interventions enregistrées.

---

## 3. Liste des clients

Cette page permettra de consulter les clients enregistrés.

Informations prévues :

- nom ;
- prénom ou raison sociale ;
- téléphone ;
- email ;
- adresse.

---

## 4. Fiche client

Cette page permettra de consulter les informations détaillées d’un client.

Elle pourra aussi afficher les chantiers associés à ce client.

---

## 5. Liste des chantiers

Cette page affichera les chantiers enregistrés.

Informations prévues :

- nom du chantier ;
- client associé ;
- adresse ;
- statut ;
- date de début.

---

## 6. Fiche chantier

Cette page centralisera les informations principales d’un chantier.

Elle devra permettre d’accéder :

- aux informations générales ;
- aux documents ;
- aux photos ;
- aux interventions ;
- au rapport PDF.

---

## 7. Documents et photos

Cette page permettra de gérer les fichiers liés à un chantier.

Documents prévus :

- devis ;
- factures ;
- plans ;
- autres PDF.

Photos prévues :

- photos de début de chantier ;
- photos d’avancement ;
- photos de fin de chantier.

---

## 8. Interventions

Cette page permettra de suivre l’historique des actions réalisées sur un chantier.

Chaque intervention contiendra :

- une date ;
- une description ;
- un utilisateur ayant ajouté l’intervention.

---

## 9. Rapport PDF

Cette page permettra plus tard de générer un rapport simple de chantier.

Le rapport pourra contenir :

- les informations du chantier ;
- le client ;
- l’historique ;
- les documents ;
- les photos.

---

## 10. Administration utilisateurs

Cette page sera réservée à l’administrateur.

Elle permettra plus tard de gérer :

- les utilisateurs ;
- les rôles ;
- les accès.

---

## Limite

Ces pages sont uniquement prévues à cette étape.

Elles seront développées progressivement lorsque les compétences HTML, CSS, SQL et PHP auront été étudiées.
